4.1.2 Memory Hierarchy
───────────────────────────────────────────────────────

🔑 GOAL:
“To obtain the highest possible access speed while minimizing the total cost of the memory system.”

• The memory unit that communicate directly with CPU is called **Main Memory**.  
• Devices that provides backup storage are called **Auxiliary Memory**.  
• Only program and data currently needed by the processor reside in main memory.  
• The Main Memory communicate directly with CPU and with Auxiliary Memory Devices through an I/O Processor.  
• Cache is used to increase the speed of processing by making current programs and data available to CPU.  
• Auxiliary Memory has Large storage capacity, Relatively Inexpensive, Low access speed compared to main memory.  
• Cache Memory has Very small, Relatively expensive, very high access speed.  

### Advantage of Memory Hierarchy:
• Increase capacity  
• Improve average access time  
• Decrease cost/bit  
• Decrease frequency of accesses to slow memory

4.2.1 Various Types of Main Memories
───────────────────────────────────────────────────────

● RAM (Random Access Memory)

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: RAM                              │
│ DEFINITION: Random Access Memory       │
│ CONTEXT: Main memory                   │
│ EXAMPLE: SRAM, DRAM                    │
└─────────────────────────────────────────┘

• RAM losses its contents when power is turned OFF → **Volatile Memory**  
• User save data but may not store permanently  
• Random selection of Memory Location  
• Microprocessor can write or read  

Types:
1) Static RAM (SRAM)  
2) Dynamic RAM (DRAM)  

### Static RAM
• Consists of flip flops  
• Used in implementing cache  
• Expensive, fast, low density, high power  
• Shorter read/write cycles  

### Dynamic RAM
• Used in implementing main memory  
• Stores binary info as electric charges  
• Reduced power consumption  
• Large storage capacity


📋 TABLE 1: RAM Function Table

| CS1 | CS2 | RD | WR | Memory Function | State of Data Bus |
|-----|-----|----|----|----------------|------------------|
| 0   | 0   | X  | X  | Inhibit        | High-impedance   |
| 1   | 0   | 0  | 1  | Write          | Input data to RAM|
| 1   | 0   | 1  | 0  | Read           | Output data      |
| 1   | 1   | X  | X  | Inhibit        | High-impedance   |

• Bidirectional data bus  
• High Impedance = Open circuit  
• Each word = 8 bits  
• Requires 7-bit address  
• CS2 bar enabled when = 0  

──────────────────────────────────────
● ROM (Read Only Memory)

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: ROM                              │
│ DEFINITION: Read Only Memory           │
│ CONTEXT: Non-volatile memory           │
└─────────────────────────────────────────┘

• Does not lose contents when power OFF  
• Microprocessor can only read  
• Programmable once  

Types:
• PROM  
• EPROM  
• EEPROM  
• Masked ROM  

### PROM
• Programmable Read Only Memory  
• Program once, never change  
• Write once, read many times  
• Poly silicon wires matrix  
• Viewed as diodes/fuses  

### EPROM
• Stores bit by charging gate of FET  
• UV light erases data  
• Reprogrammable  
• Erasing takes 15–20 minutes  

### EEPROM
• Electrically erasable  
• Erase entire chip in 10ms  
• More expensive than EPROM  

──────────────────────────────────────
● Associative Memory (CAM)

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: Associative Memory               │
│ DEFINITION: Memory accessed by content │
│ CONTEXT: CAM                           │
└─────────────────────────────────────────┘

Search Process:
1. Choose sequence of addresses  
2. Read content  
3. Compare with searched items  

• No address given during write  
• Finds empty location automatically  
• More expensive than RAM  
• Argument register A & Key register = n bits  
• Match register M = m bits  

💡 EXAMPLE 1:

A = 101 111100  
K = 111 000000  

Word1 = 100 111100 → No match  
Word2 = 101 000011 → Match Occur

4.2.2 Various Types of Auxiliary Memories
───────────────────────────────────────────────────────

● Magnetic Tape
• Magnetic coating on thin plastic strip  
• Secondary Storage  
• Length: 600–3000 ft  
• Packaged as:
  - Open Reel  
  - Cartridge  
  - Cassette  
• RW heads mounted on each track  
• Stored in blocks (records)  
• Each record has identification bit pattern at beginning and end  

● Floppy Disk
• Removable disk  
• Plastic coated magnetic material  

Types:
• 5¼ inch: 100k – 1.2MB  
• 3½ inch: 1.44MB  

• Slower access than Hard Disk  
• Less storage, less expensive, portable  

● Hard Disk
• Bits stored in concentric Tracks  
• Tracks divided into Sectors  
• Address includes:
  - Disk number  
  - Surface  
  - Sector  
  - Track  

Read/Write:
• Position head  
• Wait for sector  
• Transfer fast after reach  

● Flash Memory
• Variation of EEPROM  
• Non-volatile  
• Used in USB, SD, SSD  
• Composed of NOR or NAND gates  
• Erase at block level  
• EEPROM erase at byte level  

Performance Parameters:
• Access Time  
• Transfer Time  
• Transfer Rate

4.2.3 Cache Memory
───────────────────────────────────────────────────────

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: Cache Memory                     │
│ DEFINITION: Small, temporary fast      │
│ memory between CPU and main memory     │
└─────────────────────────────────────────┘

• Access time less than main memory  
• Speed similar to CPU registers  

Operation:
• CPU checks cache  
• HIT → read from cache  
• MISS → read from main memory and transfer block  

🔢 FORMULA 1:
Hit Rate = No. of Hits / (Hits + Misses)

🔢 FORMULA 2:
Miss Rate = 1 – Hit Rate  

Writing into Cache:
Read → Main memory not involved  
Write → Main memory involved  

Write Through:
• Update cache & main memory simultaneously  

Write Back:
• Update cache only  
• Mark by flag  
• Copy to memory when removed  

Mapping Types:
1) Associative Mapping  
2) Direct Mapping  
3) Set Associative Mapping  

CPU sends 15-bit address  
32 × 1024 = 2¹⁵  
If hit → CPU accepts 12-bit data  

### Associative Mapping
• Stores address + data  
• Any cache location store any word  
• 15-bit shown as 5-digit octal  

### Direct Mapping
15-bit divided into:
Tag (6 bits) | Index (9 bits)

### Set Associative Mapping
• Each index refers to 2 data words  
• Tag = 6 bits  
• Data = 12 bits  
Word length = 2 × (6+12) = 36 bits  
Cache size = 512 × 36

4.2.4 Virtual Memory
───────────────────────────────────────────────────────

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: Virtual Memory                   │
│ DEFINITION: Illusion of very large     │
│ memory system                          │
└─────────────────────────────────────────┘

• Virtual Address → Used by programmer  
• Address Space → Set of Virtual Address  
• Physical Address → In Main Memory  
• Memory Space → Set of Physical Address  

Example:
Main Memory: 32K  
32 × 1024 = 2¹⁵  
Physical Address = 15 bits  

Auxiliary Memory: 1024K  
1024 × 1024 = 2²⁰  
Virtual Address = 20 bits  

Address field = 20 bits  
Physical memory = 15 bits  
Mapping table required
