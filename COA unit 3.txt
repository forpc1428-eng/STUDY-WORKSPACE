1. Instruction Format and Opcode Format of 8085 μP
═══════════════════════════════════════════════

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: Instruction                      │
│ DEFINITION: An instruction is a binary │
│ code that specifies a particular       │
│ operation to be performed by the       │
│ microprocessor.                        │
│ CONTEXT: 8085 Instruction Set          │
│ EXAMPLE: ADD B, MOV A, C               │
└─────────────────────────────────────────┘

🔑 Each instruction of 8085 has **1 byte opcode**.  
With 8-bit binary code → 2⁸ = **256 different binary codes** possible.  
Out of these, **246 codes are used for opcodes**.

### Instruction Size Classification

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: Instruction Size                 │
│ DEFINITION: Number of bytes occupied   │
│ by an instruction in memory.           │
│ TYPES: 1-byte, 2-byte, 3-byte          │
└─────────────────────────────────────────┘

#### 1-Byte Instruction
• Contains opcode only  
Example: CMA, MOV A,B  

#### 2-Byte Instruction
• Opcode + 8-bit data/address  
Example: MVI B, 3EH  

#### 3-Byte Instruction
• Opcode + 16-bit data/address  
• Storage sequence in memory:  
  1. Opcode  
  2. Low byte of address/data  
  3. High byte of address/data  

📊 DIAGRAM: Instruction Format

One Byte:
┌────────┐
│ OPCODE │
└────────┘

Two Bytes:
┌────────┐ ┌────────────┐
│ OPCODE │ │ 8-bit Data │
└────────┘ └────────────┘

Three Bytes:
┌────────┐ ┌────────────┐ ┌────────────┐
│ OPCODE │ │ Low Byte   │ │ High Byte  │
└────────┘ └────────────┘ └────────────┘

2. Addressing Modes of 8085
═══════════════════════════════════════════════

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: Addressing Mode                  │
│ DEFINITION: The method of specifying   │
│ data into the instruction.             │
│ CONTEXT: Data may be in register,      │
│ memory, or immediate in instruction.   │
└─────────────────────────────────────────┘

### Types of Addressing Modes (5)

1. Immediate Addressing Mode  
2. Register Addressing Mode  
3. Direct Addressing Mode  
4. Indirect Addressing Mode  
5. Implied Addressing Mode  

---

### 1. Immediate Addressing Mode
• Data is part of instruction  
• Mnemonics contain ‘I’  

Example:  
MVI B, 3EH → Move 3EH into B  

---

### 2. Register Addressing Mode
• Register name specifies data location  

Example:  
MOV A, B  

---

### 3. Direct Addressing Mode
• 16-bit memory address given directly  

Example:  
LDA 1050H  

---

### 4. Indirect Addressing Mode
• Address stored in register pair (HL)  
• Identified by letter ‘M’  

Example:  
MOV A, M  

---

### 5. Implied Addressing Mode
• No operand  

Example:  
CMA

═══════════════════════════════════════════════
4. Data Transfer Instructions
═══════════════════════════════════════════════

🔑 Purpose: Copy data from source to destination.  
Source remains unchanged.

### MOV
MOV Rd, Rs  
Example: MOV B, C  

### MVI
MVI Rd, data  
Example: MVI B, 57H  

### LDA
LDA 16-bit address  
Example: LDA 2034H  

### LDAX
LDAX B/D  
Example: LDAX B  

### LXI
LXI H, 2034H  

### LHLD
LHLD 2040H  

### STA
STA 4350H  

### STAX
STAX B  

### SHLD
SHLD 2470H  

### XCHG
Exchange HL with DE  

### SPHL
Copy HL to SP  

### XTHL
Exchange HL with top of stack  

### PUSH
PUSH B  

### POP
POP H  

### OUT
OUT F8H  

### IN
IN 8CH

9. Stack
═══════════════════════════════════════════════

📘 DEFINITION:

• Address stored in SP  
• Follows LIFO  

Example:
LXI SP, 2099H  
LXI H, 42F2H  
PUSH H  
POP H

12. Interrupts in 8085
═══════════════════════════════════════════════

📘 DEFINITION:
Interrupt means interrupting normal execution.

Sequence:
1. Push PC  
2. Load vector address  
3. Execute ISR  
4. RET  

### Hardware Interrupts
TRAP  
RST 7.5  
RST 6.5  
RST 5.5  
INTR  

### Classification
• Maskable: RST 7.5, 6.5, 5.5, INTR  
• Non-Maskable: TRAP  
• Vectored: TRAP, RST 7.5, 6.5, 5.5  
• Non-Vectored: INTR  
• Edge Triggered: RST 7.5  
• Level Triggered: RST 6.5, RST 5.5, INTR  
• Edge & Level: TRAP  

### Priority

| Interrupt | Priority |
|-----------|----------|
| TRAP | 1 |
| RST 7.5 | 2 |
| RST 6.5 | 3 |
| RST 5.5 | 4 |
| INTR | 5 |

═══════════════════════════════════════════════
13. 8085 Vectored Interrupts
═══════════════════════════════════════════════

TRAP → 0024H  
RST 7.5 → 003CH  
RST 6.5 → 0034H  
RST 5.5 → 002CH  

INTR → Non-vectored (address provided externally)

• TRAP is non-maskable  
• Others maskable via SIM  
• Controlled using EI and DI
