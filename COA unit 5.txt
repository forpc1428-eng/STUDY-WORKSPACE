5.1 Explain I/O Interface
───────────────────────────────────────────────────────

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: I/O Interface                    │
│ DEFINITION: Hardware unit that         │
│ connects CPU with I/O devices through  │
│ registers and control circuitry.       │
│ CONTEXT: Input-Output Organization     │
└─────────────────────────────────────────┘

• It consists of two data registers called ports, a control register, a status register, bus buffers, and timing and control circuit.  
• The interface communicates with the CPU through the data bus.  
• The chip select and register select inputs determine the address assigned to the interface.  
• The I/O read and writes are two control lines that specify an input or output, respectively.  
• The four registers communicate directly with the I/O device attached to the interface.  
• A command is passed to the I/O device by sending a word to the appropriate interface register.  
• The control register receives control information from the CPU.

📝 Components:

1. **Port A Register**
   • Data register  
   • Can be input port  

2. **Port B Register**
   • Data register  
   • Can be output port  

3. **Control Register**
   • Receives control info from CPU  

4. **Status Register**
   • Holds status flags  

5. **Bus Buffers**
   • Connect interface to CPU  

6. **Timing & Control**
   • Generates timing signals  

────────────────────────────────────────
📋 TABLE 1: Register Selection

| CS | RS1 | RS0 | Register Selected |
|----|-----|-----|------------------|
| 0  | X   | X   | None: data bus in high impedance |
| 1  | 0   | 0   | Port A register |
| 1  | 0   | 1   | Port B register |
| 1  | 1   | 0   | Control register |
| 1  | 1   | 1   | Status register |

📊 TABLE ANALYSIS:
• CS = Chip Select  
• RS1, RS0 select one of four registers  
• X means don’t care  
• High impedance = bus disconnected  

Additional Points:
• Port A may be input and Port B output.  
• Interface registers communicate via bidirectional data bus.  
• RS1 and RS0 connected to least significant address lines.  
• Content transferred to CPU when I/O read enabled.  
• CPU writes data when I/O write enabled.

5.2 Differentiate various Modes of Data Transfer with I/O
───────────────────────────────────────────────────────

Data transfer handled in three modes:

1) Programmed I/O  
2) Interrupt initiated I/O  
3) Direct Memory Access (DMA)  

──────────────────────────────
### 1) Programmed I/O

• I/O device does not have direct access to memory.  
• Transfer requires execution of several instructions by CPU.  

📊 DIAGRAM 2: Programmed I/O Data Path

```
I/O Device → Interface → CPU → Memory
```

Operational Steps:
• Device places byte in I/O bus and enables data valid line.  
• Interface accepts byte into data register and enables data accepted line.  
• Interface sets flag bit (F) in status register.  
• Device disables data valid line.  
• CPU reads status register and checks flag.  
• If flag = 1, CPU reads data register.  
• Flag cleared to 0.  
• Interface disables data accepted line.  

──────────────────────────────
### 2) Interrupt Initiated I/O

• CPU stays in loop in programmed I/O.  
• Keeps processor busy without need.  
• Avoided by interrupt facility.  
• Interface issues interrupt request when data available.  
• CPU executes other program meanwhile.  
• Upon interrupt:
   - CPU stops task  
   - Branches to service program  
   - Processes I/O  
   - Returns to original task  

──────────────────────────────
### 3) DMA (Direct Memory Access)

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: DMA                              │
│ DEFINITION: Direct transfer of data    │
│ between I/O device and memory without  │
│ CPU intervention.                      │
└─────────────────────────────────────────┘

• Limited by CPU speed.  
• Remove CPU from path.  
• DMA controller manages buses.  

Control Signals:
• BR (Bus Request)  
• BG (Bus Grant)  

Operation:
• DMA sends BR.  
• CPU places address, data, read, write into high impedance.  
• CPU activates BG.  
• DMA takes control and transfers data.  
• DMA disables BR.  
• CPU disables BG.  

📊 DIAGRAM 3: DMA Bus Control

```
DMA → BR → CPU
CPU → BG → DMA

When BG = 1:
Address bus → High impedance
Data bus → High impedance
RD/WR → High impedance
```

DMA Registers:
• Selected by DS and RS  
• RD and WR are bidirectional  
• When BG=0 → CPU accesses DMA registers  
• When BG=1 → DMA communicates directly with memory  
• Word count register holds number of words

5.3 Describe Input-Output Processor (IOP)
───────────────────────────────────────────────────────

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: Input-Output Processor (IOP)     │
│ DEFINITION: External processor with    │
│ DMA capability that communicates       │
│ directly with I/O devices.             │
└─────────────────────────────────────────┘

• Instead of each interface communicating with CPU, system may have external processor.  
• Each processor communicates directly with I/O devices.  
• IOP classified as processor with DMA capability.  
• Provides path for transfer between peripherals and memory.  
• Operates independent of CPU.

5.4 Describe CPU–IOP Communication
───────────────────────────────────────────────────────

📊 DIAGRAM 5: CPU–IOP Operation Flow

```
CPU Operations                  IOP Operations

Send test instruction  →        Transfer status word
If status OK → start I/O →      Access memory for IOP program
CPU continues other program     Conduct I/O transfer
Request IOP status              I/O transfer complete → Interrupt CPU
Check status word               Transfer status word to memory
Continue
```

Operational Steps:

• CPU sends instruction to test IOP path.  
• IOP inserts status word in memory.  
• Status word indicates:
   - IOP overload condition  
   - Device busy  
   - Device ready  
• CPU checks status word.  
• If status OK → send start I/O instruction.  
• CPU continues another program.  
• When IOP terminates → interrupt request to CPU.  
• CPU requests IOP status.  
• IOP sends status report.  
• CPU checks status word for correct transfer.

📚 OVERALL KEY TAKEAWAYS:
1. I/O interface connects CPU and peripherals using registers.  
2. Three data transfer modes: Programmed, Interrupt, DMA.  
3. DMA improves speed by bypassing CPU.  
4. IOP is independent processor with DMA capability.  
5. CPU and IOP communicate via status words and interrupts.
