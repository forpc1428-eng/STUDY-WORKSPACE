## 2.1 Introduction

───────────────────────────────────────────────────────

• The 8085 microprocessor is an 8-bit microprocessor introduced by Intel Corporation in 1976.

• The 8085 microprocessor is designed to execute a set of instructions to perform various tasks in a computer system.

• The 8085 microprocessor operates at a clock speed of 3 MHz which is known as frequency.

🔢 FORMULA 1: Time for One Operation

T = \frac{1}{3\text{ MHz}} = 0.33 \times 10^{-6}\text{ second} = 0.33\ \mu s

┌─────────────────────────────────────────┐
│ Where:                                  │
│ • T = Time required for one operation   │
│ • MHz = Mega Hertz                      │
│                                          │
│ Application: Used to calculate the time │
│ required for one operation execution.   │
│                                          │
│ Final Value: 0.33 microsecond           │
└─────────────────────────────────────────┘

• To understand the basic concepts of microprocessor device, it is easier to learn from a simple 8-bit processor than from a 64-bit processor.

• A computer with a microprocessor is known as a microcomputer.

───────────────────────────────────────────────────────

📘 DEFINITION:
┌─────────────────────────────────────────┐

There are three types of system bus in 8085 microprocessor:

1. Address Bus
2. Data Bus
3. Control Bus

## Address Bus

───────────────────────────────────────────────────────

📘 DEFINITION:
┌─────────────────────────────────────────┐

│ CONTEXT: Used during read/write         │
│ operations.                             │
└─────────────────────────────────────────┘

• Size of address bus is 16 bits.

🔢 FORMULA 2: Addressable Memory Calculation

2^{16} = 64\times1024 = 64K\text{ bytes}

⚠️ IMPORTANT NOTE:
8085 can address 64 KB memory because of its 16-bit address bus.

## Address Bus Pins

───────────────────────────────────────────────────────

• Consists of 8 pins (A15 to A8).

• 8085 has 16 signal lines used as address.

• Address lines split into two groups:

* A15 to A8
* AD7 to AD0

• A15 to A8 are unidirectional and used for higher order address bus.

• AD7 to AD0 used for dual purpose.

───────────────────────────────────────────────────────

## Data Bus / Multiplexed Address Bus

───────────────────────────────────────────────────────

• Consists of 8 pins (AD7 to AD0).

• 8085 has 8 data lines (D7–D0).

• Data lines multiplexed with lower order address lines (A7–A0).

• Multiplexing reduces number of pins.

• AD7 to AD0 are bidirectional.

⚠️ IMPORTANT NOTE:
AD7–AD0 carry address during T1 state and data during remaining states.

───────────────────────────────────────────────────────

## S1 and S0 Signals

───────────────────────────────────────────────────────

• Identify operations like:

* Opcode fetch
* Memory read/write
* I/O read/write
* Interrupt acknowledgment

───────────────────────────────────────────────────────

## Power Supply and Frequency Signals

───────────────────────────────────────────────────────

• Includes:

* Vcc
* Vss
* CLK OUT
* X1
* X2

• 8085 requires +5V power supply.

• Vss connected to ground.

• CLK OUT used as system clock for other devices.

• Crystal connected between X1 and X2 generates 6 MHz frequency.

───────────────────────────────────────────────────────

## Externally Initiated Signals

───────────────────────────────────────────────────────

• Includes interrupt pins:

* TRAP
* RST 7.5
* RST 6.5
* RST 5.5
* INTR
* INTA

───────────────────────────────────────────────────────
📋 TABLE 1: Interrupt Characteristics

| Interrupt | Type                   | Priority | Vector Address    |
| --------- | ---------------------- | -------- | ----------------- |
| TRAP      | Non-maskable           | Highest  | 0024H             |
| RST 7.5   | Maskable               | High     | 003CH             |
| RST 6.5   | Maskable               | Medium   | 0034H             |
| RST 5.5   | Maskable               | Low      | 002CH             |
| INTR      | Maskable, Non-vectored | Lowest   | External Hardware |

📊 TABLE ANALYSIS:
• Purpose: Shows interrupt hierarchy of 8085
• Key Patterns: TRAP has highest priority
• Important Values: Vector addresses of interrupts
• Takeaways: Interrupt priority controls response order

⚠️ IMPORTANT NOTE:
TRAP interrupt can never be ignored by microprocessor.

───────────────────────────────────────────────────────

## DMA Transfer Signals

───────────────────────────────────────────────────────

### HOLD

• Indicates another device wants to use buses

### HLDA

• Acknowledges transfer of control of buses

───────────────────────────────────────────────────────

## READY Signal

───────────────────────────────────────────────────────

• Synchronizes operations with slower devices

• Low READY signal places microprocessor into wait state

───────────────────────────────────────────────────────

## RESET Signals

───────────────────────────────────────────────────────

### RESET IN

• Active low input signal
• Resets microprocessor

### RESET OUT

• Active high output signal
• Resets peripheral devices

───────────────────────────────────────────────────────

## Serial I/O Signals

───────────────────────────────────────────────────────

### SID

• Serial Input Data

### SOD

• Serial Output Data

🔬 PRACTICAL APPLICATION:
Used in telephone lines and serial printers.

───────────────────────────────────────────────────────

───────────────────────────────────────────────────────

───────────────────────────────────────────────────────

### Accumulator (A)

• 8-bit register
• Default register for arithmetic and logic operations

### General Purpose Registers

• B, C, D, E, H, L
• Work as pairs:

* BC
* DE
* HL

### Program Counter

• 16-bit register
• Stores address of next instruction

### Stack Pointer

• 16-bit register
• Stores address of top of stack

### Flag Register

• Contains flags used after operations

## 2.4 Register Organization of 8085

═══════════════════════════════════════════════

• Registers used for temporary storage and manipulation of data and instructions.

• 8085 contains both 8-bit and 16-bit registers.

• Registers classified into:

1. General Purpose Registers
2. Special Purpose Registers

───────────────────────────────────────────────────────

## General Purpose Registers

───────────────────────────────────────────────────────

• Six registers:

* B
* C
* D
* E
* H
* L

• Register pairs:

* BC
* DE
* HL

⚠️ IMPORTANT NOTE:
In register pair, left register stores MSB and right register stores LSB.

💡 EXAMPLE 1: LXI Instruction

┌─────────────────────────────────────────┐
│ Instruction: LXI H, 2030H              │
│                                          │
│ Operation:                               │
│ • 20H stored into H register            │
│ • 30H stored into L register            │
│                                          │
│ KEY LEARNING:                           │
│ HL register pair stores 16-bit data     │
└─────────────────────────────────────────┘

• Register C generally used as counter register.

───────────────────────────────────────────────────────

## Special Purpose Registers

───────────────────────────────────────────────────────

### Accumulator (A)

• 8-bit register
• Used in arithmetic, logic, I/O and LOAD/STORE operations
• Result stored in accumulator

### Program Counter (PC)

• 16-bit register storing address of next instruction

🔢 FORMULA 3: PC Increment Logic

PC = PC + n

┌─────────────────────────────────────────┐
│ Where:                                  │
│ • n = instruction size in bytes         │
│                                          │
│ Cases:                                  │
│ • One-byte instruction → n = 1          │
│ • Two-byte instruction → n = 2          │
│ • Three-byte instruction → n = 3        │
└─────────────────────────────────────────┘

### Stack Pointer (SP)

• 16-bit memory pointer
• Points to stack memory location

📘 DEFINITION: Stack
┌─────────────────────────────────────────┐
│ TERM: Stack                             │
│ DEFINITION: Portion of RAM used for     │
│ temporary storage of return addresses.  │
└─────────────────────────────────────────┘

───────────────────────────────────────────────────────

## Flag Register

───────────────────────────────────────────────────────

• 8-bit register
• Only 5 bits are useful

### Sign Flag (SF)

• Set to 1 if result negative
• Set to 0 if result positive

### Zero Flag (ZF)

• Set to 1 if result zero
• Set to 0 if result non-zero

### Auxiliary Carry Flag (AC)

• Set when carry generated from 4th bit to 5th bit

### Parity Flag (PF)

• Set to 1 for even parity
• Reset to 0 for odd parity

### Carry Flag (CF)

• Set to 1 when carry from MSB occurs

## 2.5 8085 Instruction Execution

═══════════════════════════════════════════════

• Programming in 8085 is called assembly language programming.

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: Assembly Language                 │
│ DEFINITION: Low-level programming       │
│ language used to communicate directly   │
│ with hardware.                          │
└─────────────────────────────────────────┘

• Assembly language converted into machine code using assembler.

───────────────────────────────────────────────────────

## Instruction

───────────────────────────────────────────────────────

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: Instruction                       │
│ DEFINITION: Command given to            │
│ microprocessor to perform operation.    │
└─────────────────────────────────────────┘

Instruction contains two parts:

1. Opcode
2. Operands

### Opcode

• Specifies operation to be performed
• Also called mnemonic code

### Operands

• Specifies data on which operation performed

───────────────────────────────────────────────────────

## Instruction Cycle

───────────────────────────────────────────────────────

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: Instruction Cycle                 │
│ DEFINITION: Time taken to complete      │
│ execution of instruction.               │
└─────────────────────────────────────────┘

• Consists of one to six machine cycles.

───────────────────────────────────────────────────────

## Machine Cycle

───────────────────────────────────────────────────────

• Operations performed sequentially during instruction execution

Examples:

* Opcode fetch
* Memory read
* Memory write
* I/O read
* I/O write

• Machine cycle consists of 3 to 6 T-states.

───────────────────────────────────────────────────────

## T-State

───────────────────────────────────────────────────────

📘 DEFINITION:
┌─────────────────────────────────────────┐
│ TERM: T-State                           │
│ DEFINITION: One clock pulse used for    │
│ synchronization of operations.          │
└─────────────────────────────────────────┘

───────────────────────────────────────────────────────

## Execution Steps of Instruction Cycle

───────────────────────────────────────────────────────

### 1. Fetch Cycle

• Fetch opcode from memory using PC
• Store opcode into instruction register

### 2. Decode Instruction

• Instruction decoder interprets opcode

### 3. Find Effective Address

• Fetch required data from memory

### 4. Execution Cycle

• Perform operation specified by opcode

Includes:

* MEMR
* MEMW
* IOR
* IOW

───────────────────────────────────────────────────────
📊 DIAGRAM 4: Instruction Cycle
───────────────────────────────────────────────────────

🎨 ASCII/TEXT RECREATION:

```text id="w4l82l"

        │
        ▼
   DECODE
        │
        ▼
 FIND EFFECTIVE
     ADDRESS
        │
        ▼

```

📝 DETAILED EXPLANATION:

• Fetch obtains instruction
• Decode identifies operation
• Address stage obtains operand
• Execute performs operation

⚠️ IMPORTANT NOTE:
Fetch cycle takes 4 T-states and execution cycle takes 3 T-states.

📚 OVERALL KEY TAKEAWAYS:

1. 8085 is an 8-bit Intel microprocessor operating at 3 MHz frequency.
2. System bus consists of Address Bus, Data Bus and Control Bus.
3. 8085 pin diagram signals are classified into six major groups.
4. Interrupt system contains five interrupts with TRAP as highest priority.
5. Register organization includes general purpose and special purpose registers.
6. Instruction execution consists of fetch, decode, address and execute phases.
